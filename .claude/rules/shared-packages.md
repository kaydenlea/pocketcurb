# Shared Packages Rule

Shared packages exist for domain logic, schemas, contracts, types, and engineering standards.

## Put in Shared Packages

- business rules
- shared validation
- API contracts
- Supabase types
- cross-lane config

## Do Not Put in Shared Packages

- mobile-only UI flows
- web-only UX or SEO structure
- lane-specific product decisions disguised as generic abstractions
