# Mobile Rule

Treat `apps/mobile` as the primary product lane. Optimize for daily clarity, Safe-to-Spend trust, low-friction transaction handling, reimbursement correctness, and privacy-first shared finance.

## Guardrails

- keep mobile UX distinct from web SEO needs
- use shared packages for business logic, not mobile UI decisions
- treat secure storage and auth flows as security-sensitive work
