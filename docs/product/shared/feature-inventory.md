# Feature Inventory

## MVP Differentiation Focus

The MVP should not position every feature as equally important. The primary differentiating story should be:

1. a trusted short-term clarity hub
2. event-aware budgeting that becomes a shareable artifact
3. place-aware transaction context that turns spend into memory and recommendations
4. shared-spend and reimbursement handling that does not distort the truth

The strongest wedge is not "social budgeting" in the abstract. The stronger wedge is a decision-first money product that can also produce event receipts, weekly stories, and spending maps worth keeping or sharing.

The MVP should optimize for four product truths:

- users need a fast first answer before they care about richer insight
- the product needs a repeated overview or event habit loop, not a broad feature checklist
- social should deepen usefulness and referrals, not create pressure to perform spending publicly
- map and event layers should reduce fragmentation between finance, group chat, and place-memory workflows

And three implementation truths:

- frictionless capture matters from the start, even if voice is deferred
- core resilience matters from the start, even if full offline or local-first modes are deferred
- the MVP should infer, prefill, or suggest before it asks the user to configure, categorize, or curate manually

## MVP Features

1. Overview page with Safe-to-Spend, category budgets, recent transactions inbox, event timeline, and upcoming transactions.
2. Cash Flow page with net flow, inflow versus outflow, and short-term forecast.
3. Bills and Recurring page with settled, due soon, and unresolved items.
4. Accounts page with balances, assets, debts, and account health overview.
5. Transactions page with chronological, category, and event views plus map toggles where data quality permits.
6. Global Quick Add for transactions, events, reimbursements, or notes.
7. Event list and Event Details page with event budget, top categories, collaborator state, and map view.
8. Automatic event grouping plus ambiguity review.
9. Curated digital receipt generation from event transactions.
10. Share links or export cards for event receipts and summaries.
11. Private-by-default spending map and favorite spots by category.
12. Split and reimbursement-aware transaction handling.
13. Privacy controls for shared contexts and social artifacts.
14. Weekly private money story with quick actions to adjust budget, review drift, or revisit events.
15. Proactive warnings and suggestions on clarity surfaces when short-term cash-flow, bill pressure, or event overspend changes materially.

## Later Features

1. Public profiles, follows, comments, and broader community graph.
2. Merchant or creator partnerships tied to places or recommendations.
3. Multi-currency support.
4. Voice or whisper-style transaction capture.
5. Budget modes and configurable guidance profiles, including FIRE-oriented modes.
6. Long-term goal and FIRE consequence views.
7. Investments and asset-planning depth.
8. Deeper running-balance analytics and dedicated balance forecasting views.
9. Freelancer or entrepreneur mode with personal vs business separation.
10. Tax-bracket projections and quarterly estimate support.
11. Privacy-first local-storage mode with strong manual entry and text or SMS transaction capture options.
12. Spreadsheet import and export.
13. Full social recommendation graph with saved guides, ranked lists, or creator subscriptions.
14. Monetized recommendation or affiliate layers.

## Feature Ordering Logic

The first release sequence should prioritize features that reduce daily friction, improve trust, and create memorable reuse before expanding into broader social or planning scope.

That means the MVP should lead with:

- daily decision support that helps users know what is safe to spend now
- onboarding that reaches a first useful answer quickly and explains why it is trustworthy
- low-friction capture and correction paths that reduce user effort from the first release
- inferred obligations, suggested event grouping, and prefills before deep manual setup
- shared-spend and reimbursement correctness that removes real-world admin pain
- event-aware organization that can become a polished digital receipt
- place-aware transaction views that make recommendations and memory retrieval easier

The MVP should not lead with:

- a public feed
- leaderboards or performative gamification
- merchant monetization
- broad community features that weaken privacy or confuse the core clarity promise
