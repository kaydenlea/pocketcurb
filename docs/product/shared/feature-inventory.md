# Feature Inventory

## MVP Differentiation Focus

The MVP should not position every included feature as equally important. The primary differentiating story should be short-term clarity with less admin work:

1. trusted Safe-to-Spend and Daily Spending Meter guidance
2. forward-looking cash-flow visibility with lightweight running-balance awareness
3. shared-spend and reimbursement handling that does not distort the truth

Automatic event grouping should be treated as a signature secondary differentiator. It is valuable because it reduces categorization work and adds narrative context in a way many mainstream budgeting apps do not emphasize, but it should strengthen the core clarity story rather than replace it.

The MVP should also optimize hard for two product truths:

- users need a fast first answer before they care about richer insights
- the product needs a repeated daily or weekly habit loop, not a broad feature checklist

## MVP Features

1. Daily Spending Meter.
2. Add or Simulate Transaction.
3. Daily Budget Setting.
4. Today's Transactions List.
5. Lightweight running-balance awareness on primary decision surfaces.
6. Event or Trip Budget with automatic event grouping.
7. Auto-assign transactions to an event, allow edits, and flag ambiguous cases.
8. Split option for shared transactions plus reimbursement-aware handling.
9. Forward-looking week or month view for insights, budgets, payments, and cash-flow visibility.
10. Safe-to-Spend number.
11. Privacy toggle on shared categories or contexts.
12. Rollover budgeting.
13. Proactive warnings and suggestions on daily guidance surfaces when short-term cash-flow or spending pressure changes materially.

## Later Features

1. Multiple currencies.
2. Voice or whisper-style transaction capture.
3. Budget modes and configurable guidance profiles, including FIRE-oriented modes.
4. Gap-fill behavior that connects FIRE planning to daily budgeting.
5. Real-time long-term impact and FIRE-date movement.
6. Exportable reports to Excel or Sheets.
7. Shared use for couples, families, or multiple people.
8. Investments and assets.
9. Deeper running-balance analytics and dedicated balance forecasting views.
10. Long-term goal framing for daily purchases.
11. Freelancer or entrepreneur mode with personal vs business separation.
12. Tax-bracket projections and quarterly estimate support.
13. Privacy-first local-storage mode with strong manual entry and text or SMS transaction capture options.
14. Spreadsheet import.
15. HYSA rate tracking and suggested cash moves.
16. Transaction export and customizable reports.

## Feature Ordering Logic

The first release sequence should prioritize features that reduce daily friction, improve trust, and fix common spreadsheet failure modes before expanding into advanced planning.

That means the MVP should lead with:

- daily decision support that helps users know what is safe to spend now
- onboarding that reaches a first useful answer quickly and explains why it is trustworthy
- shared-spend and reimbursement correctness that removes real-world admin pain
- event-aware organization as a lower-admin, more narrative alternative to category cleanup
