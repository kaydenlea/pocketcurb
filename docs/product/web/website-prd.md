# Website PRD

## Goal

The web lane exists to build trust, explain the product thesis, capture waitlist demand, and support future SEO or educational content. It is not the primary product surface.

## Web Jobs

- explain why PocketCurb is different
- convert interested visitors into waitlist signups
- create privacy and trust confidence
- support later SEO content without distorting the mobile roadmap

## Core Web Surfaces

- landing page
- waitlist page and thank-you flow
- privacy and disclosure pages
- later educational or comparison content

## Messaging Constraints

The website must stay faithful to the actual product thesis: admin work elimination, forward-looking cash flow, reimbursements and shared spending correctness, Safe-to-Spend, privacy-first resilience, Clarity-as-a-Service, event or narrative intelligence, and the later Crisis Cushion framing.
