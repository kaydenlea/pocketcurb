# Architecture Decision Records

This directory stores cross-cutting decisions that should survive individual tasks and contributors.

## When to Add an ADR

Add or update an ADR when a decision materially changes:

- monorepo structure
- platform or package boundaries
- backend or auth ownership
- storage strategy
- agent workflow expectations
- release or security posture

## ADR Expectations

- state the context clearly
- record the decision
- explain consequences and tradeoffs
- update linked docs when the decision changes
